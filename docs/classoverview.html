Below you can find the appropriate description of these libraries.

 

JDFToolsLib

This library of packages contains tools to provide support for JDFWrapperLib and JDFWrapperCoreLib libraries.

JDFToolsLib has the following structure:
  • io - Provides the classes for system input and output through data streams, serialization and the file system
  • net - Provides the classes for implementing networking applications.
  • util - Contains different utility classes (such as string tokenizer), classes for RegularExpressions support.
  • mime - Provides the classes for MIME support
  • lang - Provides the fundamental classes for C++ language such as WStirng, vectors (vint, vdouble, vWString...), maps

JDFWrapperCoreLib

This library contains packages to provide support for defined in JDF data types (such as Shape, Rectangle, various lists, rangelists) and the core packages such as

  • KElement - Simple wrapper around DOMElement. Additional vector functionality is also provided for tree walking
  • XMLDoc - Simple wrapper around DOMDocument. Additional io support
  • Parser


JDFWrapperCore packages wrap STL around Xerces to provide STL string support and STL vector support for tree walking. That allows using of non type-safe DOM classes, i.e. classes that access elements and attributes based on strings only. It also provides some simple io functionality. Details can be found here

JDFWrapperLib

The main library. It contains JDFWrapper packages and provides generic and typesafe support for Nodes, Messages, Resources, DeviceCapabilities and the core JDF packages. All automatically generated packages have the same grouping and are gathered into Auto package.

To see more choose any package. The following list provides a quick overview of the class tree. For a complete list, please refer to the full documentation tree.

  • JDFWrapper JDFNode
    Here you can find the packages that reflect general structure of JDF Nodes and Jobs (see JDF Specification 1.2 chapter 3). Among them are JDFNode, JDFAncestorPool, JDFCustomerInfo, JDFNodeInfo, JDFStatusPool, JDFResourcePool, JDFResourceLinkPool, JDFAuditPool
    • JDFNode
      Access to the JDF Node and its direct children. No typesafe support for specific elements
    • JDFResourcePool
      • Access to the ResourcePool and all its children. No typesafe support for specific resources
    • JDFResourceLinkPool
      Access to the ResourceLinkPool and its children. Contains routines for access to the Resources in ResourcePool described by ResourceLinks.
  • JDFWrapper JDFJMF
    The following packages provide both generic and typesafe support for Messages. All functionality is provided by JDFMessage.cpp and child classes.
    • JDFJMF
      Access to JMF Root
    • JDFMessage
      Typesafe access to the messages. Core routines for JMF messages including the 5 message families:

      JDFAcknowledge
      JDFCommand
      JDFQuery
      JDFResponse
      JDFSignal

  • JDFWrapper JDFResources
    Two packages provide typesafe access to all resources described in JDFSpecification. Additional two packages provide access to all subelements described in these Resources. All of them are based on automatically generated classes that are created from the XML Schema. All typesafe resources inherit from JDFResource. All their subelements inherit from JDFElement. Subelements are grouped into the appropriate packages
    • IntentResources
      Typesafe access to JDF Intent Resources
    • ProcessResources
      Typesafe access to JDF Process Resources
  • JDFWrapper JDFDeviceCapability
    Packages of JDFDeviceCapability are used to specify the Capabilities of various Devices. These provide the typesafe access to elements and attributes and implement DeviceCapability functionality according to JDFSpecification.
  • JDFWrapper JDFCore
    The following packages provide generic support for all Elements and Resources, but not typesafe support for specific elements. Here also you can find JDF Factory package
    • JDFElement
      Implementation of JDFElement. Provides an access to the elements and attributes of JDFElement and all classes inherited from it
    • JDFRefElement
      Implementation of JDFRefElement. Provides an access to the appropriate Element described by this RefElement
    • JDFResource
      Implementation of JDF Resource. Provides an access to JDF Resources including partition support
    • JDFResourceLink
      Implementation of JDF ResourceLink. Provides an access to JDF ResourceLinks including partition support
    • JDFFactory
      Factory of elements
  • JDFWrapper Auto
    The Auto packages are automatically generated packages and created from the XML Schema.
    • AutoNode
      Access to the automatically generated classes of JDFNode packages
    • AutoJMF
      Access to the automatically generated classes of JDFJMF packages
    • AutoResources
      Access to the automatically generated classes of JDFResources packages
    • AutoDeviceCapability
      Access to the automatically generated classes of JDFDeviceCapability packages
    • AutoCore
      Access to the automatically generated classes of JDFCore packages
