Step 2   Install external packages (LibPng, Zlib, OpenSSL, Xerces, STL)


2 b     OPENSSL
JDFTools uses OpenSSL 0.9.6 [engine] 24 Sep 2000
The sources can be found at http://www.openssl.org/source/openssl-0.9.6.tar.gz
Extract  openssl-0.9.6.tar.gz  to  .\external\openssl
Install Perl and set the search path to the perl.exe !!! Note
Then open Command shell, go to the oppenssl directory and type the following commands:
1) perl Configure VC-WIN32
# creates Makefile.ssl, crypto/opensslconf.h
2) ms\do_ms
# creates MINFO, ms\w31dll.mak, ms\nt.mak, ms\ntdll.mak, ms\libeay16.def
# ms\libeay32.def, ms\ssleay16.def, ms\ssleay32.def
3) nmake -f ms\ntdll.mak
2 c     XERCES
JDFTools uses Xerces-C++ V2.3 But V2.6 was tested too and you can use it
It can be found at http://xml.apache.org/xerces-c/index.html

This installation assumes a Xerces-C++ 32 bit Binary distribution for Windows :
http://xml.apache.org/dist/xerces-c/archives/Xerces-C_2_3_0/xerces-c_2_3_0-win32.zip     !!! Note

Extract  xerces-c_2_3_0-win32.zip  to  .\external\xerces
The Xerces DLLs (xerces-c_2_3_0.dll and for debug purpose xerces-c_2_3_0D.dll) are in external\xerces\bin. Copy both of them into the directory, where they can be available for all applications. They are needed somewhere in the search path, e.g. in Windows\system32
2 d     STL
Only for VS C++ V6.0 !
An open source version of the Standard Template Library - STL support can be found at http://www.stlport.org
Include the stlport directory in the external directory to enable STLPort. No further modifications in the VC6 dsps are necessary.
